## iPhone 快捷指令搭建指南 — Health Tracker 同步

### 前提
- iPhone 上已安装「快捷指令」App（iOS 自带）
- Health Tracker 已部署到可访问的地址（见下方部署说明）
- 记下你的部署地址，例如 `http://192.168.1.5:3000`（需替换为你实际的地址）

---

### 第 1 步：创建快捷指令

打开「快捷指令」App → 点击右上角 **+** → 点击「新建快捷指令」→ 改名「Health Tracker 同步」

### 第 2 步：添加 14 个操作

按顺序搜索并添加以下操作：

| # | 搜索关键词 | 选择的操作 | 配置参数 |
|---|-----------|-----------|----------|
| 1 | 「健康」 | **查找健康样本** | 类型=体重, 排序=开始日期, 顺序=最新排最前, 限制=1 |
| 2 | 「健康」 | **获取健康样本的详细信息** | 样本=[步骤1], 详细信息=值 |
| 3 | 「健康」 | **获取健康样本的详细信息** | 样本=[步骤1], 详细信息=开始日期 |
| 4 | 「日期」 | **格式化日期** | 日期=[步骤3], 日期格式=自定, 格式字符串=yyyy-MM-dd |
| 5 | 「健康」 | **查找健康样本** | 类型=体脂率, 排序=开始日期, 顺序=最新排最前, 限制=1 |
| 6 | 「健康」 | **获取健康样本的详细信息** | 样本=[步骤5], 详细信息=值 |
| 7 | 「健康」 | **查找健康样本** | 类型=睡眠, 排序=开始日期, 顺序=最新排最前, 限制=1 |
| 8 | 「健康」 | **获取健康样本的详细信息** | 样本=[步骤7], 详细信息=时长 |
| 9 | 「计算」 | **计算** | 输入=[步骤8], 操作=÷, 操作数=3600 |
| 10 | 「健康」 | **查找健康样本** | 类型=活动能量, 排序=开始日期, 顺序=最新排最前, 限制=1 |
| 11 | 「健康」 | **获取健康样本的详细信息** | 样本=[步骤10], 详细信息=值 |
| 12 | 「文本」 | **文本** | 粘贴下面「JSON 模板」的内容 |
| 13 | 「网页」 | **获取 URL 的内容** | 方法=POST, URL=输入你的部署地址末尾加 /api/sync, 请求体=JSON, 值为[步骤12], 头部添加 Content-Type=application/json |
| 14 | 「脚本」 | **如果** | 拖到步骤13下方, 条件=状态码 ≠ 200 → 显示通知「同步失败」, 否则 → 无操作 |

### 第 3 步：粘贴 JSON 模板

在步骤 12 的「文本」框中粘贴以下内容：

```
{
  "records": [
    {"type":"weight","value":体重值,"unit":"kg","date":"日期","source":"Apple Health"},
    {"type":"body_fat","value":体脂率值,"unit":"%","date":"日期","source":"Apple Health"},
    {"type":"sleep","value":睡眠值,"unit":"hours","date":"日期","source":"Apple Health"},
    {"type":"active_energy","value":活动能量值,"unit":"kcal","date":"日期","source":"Apple Health"}
  ]
}
```

> ⚠️ 粘贴后，依次选中 JSON 中的「体重值」「体脂率值」「睡眠值」「活动能量值」「日期」，长按 → 选择「插入变量」→ 分别选对应的步骤2、步骤6、步骤9、步骤11、步骤4 的输出。

### 第 4 步：测试 + 自动化

1. 点击右下角 ▶ 运行一次，验证是否能成功推送数据
2. 首次运行会弹出四项健康数据授权，全部点「允许」
3. 在「自动化」标签 → 新建「个人自动化」→ 选择「特定时间」→ 每天 08:00
4. 添加操作「运行快捷指令」→ 选择「Health Tracker 同步」
5. **关闭「运行前询问」**（重要！否则每次都弹窗）
6. 点击完成

### 第 5 步：分享 iCloud 链接

快捷指令列表 → 长按「Health Tracker 同步」→ 分享 → **「拷贝 iCloud 链接」**

把生成的链接发给我或保存备用，以后重装手机可以直接导入。

## 部署地址说明

快捷指令需要在 iPhone 上通过 HTTP 访问 Health Tracker。你需要让服务可被 iPhone 访问：

**方案 A（最简单，今天可用）：**
```
1. 笔记本连 WiFi，iPhone 连同一 WiFi
2. 笔记本启动：pnpm start -p 3000 -H 0.0.0.0
3. 查笔记本 IP：ifconfig | grep "inet " | grep -v 127
4. iPhone 快捷指令 URL 填：http://192.168.x.x:3000/api/sync
```

**方案 B（外网也能用，推荐）：**
```
部署到 Railway.app（免费，支持文件持久化）：
1. 注册 railway.app → 新建项目 → Deploy from GitHub
2. 上传代码到 GitHub（不含 node_modules）
3. Railway 自动构建部署
4. 得到 https://xxx.railway.app 公网地址
5. iPhone 快捷指令 URL 填：https://xxx.railway.app/api/sync
```
